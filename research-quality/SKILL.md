# 리서치 스킬

근거 기반의 구조화된 리서치 보고서를 작성한다. 목표는 단순 요약이 아니라, 사용자가 이 보고서만으로도 실질적인 판단을 내릴 수 있게 만드는 것이다.

## 핵심 원칙

- 최소 5개 이상의 서로 다른 출처를 사용한다. 단일 출처에 의존하면 편향되거나 불완전한 결론에 이르기 쉽기 때문이다.
- 1차 출처를 우선 사용한다. 2차 출처는 원문을 요약하거나 해석하는 과정에서 맥락이 빠지거나 왜곡될 수 있다.
- 핵심 주장, 수치, 비교 포인트마다 출처 URL을 포함한다.
- 찬반, 장단점, 반론, 리스크를 함께 다룬다. 한쪽만 다루면 의사결정자가 리스크를 과소평가할 수 있다.
- 최신성이 중요한 주제는 최근 1개월 이내 자료를 우선 사용한다.
- 모르는 내용은 추측하지 말고 불확실성으로 명시한다.
- 벤더 주장이나 마케팅 문구는 가능하면 독립 출처로 교차검증한다.
- 의료, 법률, 금융 등 전문 분야는 이 보고서만으로 최종 판단을 내리기 어렵다. 해당 분야의 조사 결과에는 "전문가 추가 검증을 권장합니다"를 포함한다.

## 출처 우선순위

1. 공식 문서 / 원문 / 논문 / 벤더 공식 문서
2. 신뢰도 높은 업계 분석 / 리서치 기관 / 주요 테크 미디어
3. 전문가 블로그 / 기술 블로그
4. 커뮤니티 / 포럼

## 워크플로우

### 1. 요청 해석

먼저 아래를 파악한다.

- 조사 대상
- 조사 목적
- 비교형인지 탐색형인지
- 최신성이 중요한 주제인지
- 사용자가 직접 제시한 판단 기준이 있는지

사용자가 판단 기준을 주지 않았다면 아래 기본 기준을 사용하고, 본문에 그 가정을 명시한다.

- 성능
- 비용
- 운영 난이도
- 생태계 / 커뮤니티
- 안정성 / 리스크
- 적합한 사용 사례

비교형이면 반드시 비교표를 포함하고, 최종적으로 어떤 선택이 더 적합한지까지 정리한다.
탐색형이면 현황, 핵심 포인트, 시사점 중심으로 정리한다.

### 2. 검색 전략 수립

주제가 한국어권이어도 필요하면 한국어와 영어 키워드를 함께 사용한다.
최신성이 중요한 주제는 최근 1개월 이내 자료를 우선 찾는다.

예시:
- 제품 비교: 공식 문서 + 최근 릴리스 노트 + 독립 리뷰 + 실제 사용 사례
- 트렌드 조사: 최근 1개월 기사 + 공식 발표 + 업계 보고서
- 가격 / 정책 / 지원 여부: 가장 최신 공식 페이지 우선

### 3. 정보 수집

WebSearch로 최소 5개 이상의 서로 다른 출처를 확보한다.

- 최신성이 중요한 주제라면 핵심 근거 출처 중 가능한 한 최근 1개월 이내 자료를 포함한다.
- 출처 수를 채우기 위해 신뢰도 낮은 자료를 무리하게 늘리지 않는다.
- 신뢰할 만한 출처가 5개 미만이면, 그 사실과 이유를 보고서에 명시한다.

WebSearch를 사용할 수 없으면 학습 데이터 기반으로 작성하되, 보고서 상단에 아래 문구를 넣는다.

> 이 보고서는 실시간 웹 검색 없이 학습 데이터 기반으로 작성되었습니다. 최신 정보는 별도 확인이 필요합니다.

### 4. URL 검증

최종 보고서에 실제로 인용할 URL만 WebFetch로 검증한다.

- 검색 결과에서 직접 얻은 URL만 사용한다.
- URL을 추측하거나 조합하지 않는다.
- WebFetch로 확인되지 않으면 `[URL 미확인]`으로 표기한다.
- 일시적 오류 가능성이 있으므로, 미확인 표기는 잘못된 URL과 동일한 의미로 단정하지 않는다.

### 5. 정보 정리 및 합성

정보를 아래 기준으로 정리한다.

- 사실
- 해석
- 추천

이 셋을 섞지 말고 구분해서 작성한다.

출처 간 의견, 수치, 평가가 충돌하면 한쪽만 택하지 말고 상충 지점을 정리한다.
그 후 출처 우선순위와 최신성을 바탕으로 어느 쪽이 더 신뢰할 만한지 이유를 적는다.

### 7. 저장 전 점검

보고서를 저장하기 전에 아래를 확인한다.

- 이 보고서만으로 의사결정이 가능한가?
- 핵심 주장마다 출처 URL이 있는가?
- 결론이 본문 근거와 논리적으로 연결되는가?
- 빠뜨린 반론이나 리스크가 없는가?
- 전문 분야(의료/법률/금융 등)라면 "전문가 추가 검증 권장" 문구가 포함되었는가?

### 8. 저장

`references/config.md`에서 저장 경로를 읽고 파일을 저장한다.

**저장 절차:**
1. `references/config.md`에서 `vault_path`를 읽는다.
2. 해당 경로가 없으면 `mkdir -p`로 생성한다.
3. `YYYY-MM-DD-{주제}.md` 형식으로 저장한다. 파일명의 주제 부분은 영문으로 작성한다 (플랫폼 간 한국어 파일명 호환성 문제 방지). 본문은 한국어로 유지한다.
4. 저장에 실패하면 보고서 전문을 먼저 출력한 뒤 실패 이유를 덧붙인다.
